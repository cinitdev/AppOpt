# 更新日志

## 修复

- 完善配置并发、历史导入与更新流程稳定性。
- App 与守护进程共享 `applist.conf` 锁，并改为临时文件原子替换，避免规则互相覆盖、生成半文件或误把读取失败当成空配置写回。
- 修复历史详情异步加载中断、空记录状态不明确、读取失败后缺少重试入口的问题。
- 历史日志导入改为完整校验后再入库，损坏会话、空会话、重复会话会保留原文件并输出排查日志。
- 补充 C 端历史写入结果检查，创建、格式化、临时文件写入和原子替换失败时会输出明确警告。
- 修复前台检测 helper 重连时旧监听残留、可见任务诊断不完整以及全无信号时悬浮球残留的问题。
- 移除容易导致离开前台延迟关闭的 `stune` 前台组路径，保留 `cpuset/cpuctl` 前台组检测。
- 修复 Android 12/13 状态栏仍显示主题纯色、与渐变标题栏不一致的问题，统一使用透明状态栏并按系统状态栏高度补齐标题栏顶部间距。

## 优化

- 设置策略保存改为串行队列，并在页面离开、旋转或切到后台前提交最后一次待保存修改。
- 模块下载支持生命周期取消、30 分钟超时、任务被系统移除检测、临时文件清理和失败模块转移到 Download 目录。
- 刷入日志改为按缓冲区流式追加刷新，保留类似 Magisk/KernelSU/APatch 的连续输出效果。
- 查看规则、删除配置和规则编辑流程增加读取中状态、重复点击保护、未保存修改确认和更清晰的错误提示。
- 悬浮球触控区域扩展到 48dp，视觉尺寸保持不变，并补充标准点击语义与动态无障碍说明。
- 运行环境加入 ActivityTaskManager helper 状态，诊断包同步导出 helper 日志、状态文件和进程信息。
- 清理废弃代码资源，移除已被 Markwon 替代的旧 Markdown 轻量解析逻辑。
- 文档补充退出目标应用前台时自动停止校准并显示结果弹窗的说明。

## 自动校准

- 自动校准负载从“线程占应用总负载比例”改为按系统时钟节拍和实际采样间隔计算真实 CPU 使用率，降低轻负载线程被误判成高负载的概率。
- 主进程继续按线程记录负载；子进程改为按整体负载记录并生成进程级规则，子进程线程摘要只用于历史记录展示。
- 子进程整体负载达到策略阈值时生成 `包名:子进程=核心范围` 规则，避免生成不受 C 端解析支持的子进程线程规则。
- 为线程样本、TID、子进程线程摘要和历史曲线增加数量上限，失效 TID 自动回收，减少长时间校准的内存增长。
- 规则输出为主进程兜底规则预留空间，避免子进程规则过多时丢失主进程兜底。

## 前台检测与 FPS

- 新增常驻 root `app_process` 前台监听 helper，优先读取 ActivityTaskManager 当前任务状态。
- ActivityTaskManager 不可用、过期或报错时，自动降级到 UsageStats、cgroup 前台组和 dumpsys 焦点组合判断。
- 组合兜底改为投票逻辑，两个来源确认目标离开时关闭悬浮球；只有单一来源可用时跟随该来源。
- 悬浮球前台检测增加会话代号，旧应用的异步检测结果和延迟关闭任务不会再误关新启动的悬浮球。
- FPS 启停命令改为串行执行，切换目标应用时清理旧 FPS、旧结果浮层和旧前台检测状态。

## 构建与版本

- `build_module.sh` 编译前自动读取 `app/build.gradle.kts` 的 `versionName/versionCode`，同步 `DaemonBridge`、C 守护进程和 `magisk_module/module.prop` 版本。
- 模块构建自动编译并打包 package helper 与 ActivityTaskManager foreground helper。
- App、模块、DaemonBridge 和 C 守护进程版本统一升级到 v1.7.5（175）。
