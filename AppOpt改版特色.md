# AppOpt 增强版 — 特色说明

> 基于原版 **AppOpt v1.6.3**(作者 suto)深度扩展。原版是一个专注、可靠的 CPU 线程绑核守护进程;本增强版在其内核之上，新增了**自动校准、真实帧率监测、历史负载记录**三大守护进程能力，并从零做了一个**配套图形 App**，把命令行工具变成人人可用的一键工具。
>
> 兼容 Magisk / KernelSU / APatch，规则即时生效、无需重启。

---

## 一、守护进程（C）新增能力

原版只做"按配置文件静态绑核"。增强版让守护进程**会观察、会学习、会测帧**：

- **自动校准生成规则**
  进游戏点一下悬浮球，守护进程开始采样各线程的 CPU 负载；结束时自动按大核 / 中核 / 小核分档，生成绑核规则写回配置 —— 不用再手写 `package{thread}=cpus`。

- **真实游戏帧率监测（eBPF + SF 四档降级）**
  优先使用 **eBPF uprobe** 在内核态逐帧捕获 `libgui::queueBuffer`，精度最高；不可用时自动降级到 **SurfaceFlinger --latency** binder 直连（差分读、不清缓冲，与 Scene FAS 共存），再降级到 **--timestats**（Android 16 等兼容），最后兜底 CLI dumpsys。全程自动探测，无需手动配置。

- **历史线程负载记录（数据库存储）**
  SQLite 数据库无限保存历史（.log 循环保留 7 条作备份），FileObserver 监听文件变化零延迟导入，启动时批量导入避免卡顿。series 字段使用 Deflate 压缩，节省 60-80% 存储空间。

- **看门狗自动拉起**
  守护进程异常退出后由 service.sh 看门狗自动重启，单实例运行。

## 二、配套 App 特色（原版没有 App）

把命令行守护进程变成图形化、人人可用：

- **游戏内悬浮胶囊**：实时显示真实渲染帧率；点一下开始校准，再点结束并直接弹出生成的规则。
- **一键校准 + 自动收起**：前台监测目标应用，退出游戏即自动停止采样并收起悬浮球，不残留。
- **历史记录可视化 + 智能导入**：仿 Scene 风格折线卡片，按 ID 永久编号不受删除影响；FileObserver 监听 .log 变化零延迟导入，启动时批量导入避免卡顿。
- **运行环境自检**：Root / 悬浮窗权限 / 使用情况访问 / 守护进程存活，状态一目了然。
- **规则即时生效**：自动校准生成的规则与手写配置可并存，改完无需重启。
